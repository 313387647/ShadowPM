# ShadowPM 项目介绍

## ShadowPM 是什么

ShadowPM 是一个 AI 原生项目管控平台。

它不是任务管理工具，也不是传统项目看板。ShadowPM 解决的是更具体的问题：

> 团队手里已经有 Excel、Brief、传播日历、预算表和零散进度信息，如何快速生成一个可管理、可编辑、可追溯的项目管控工作区。

它适合轻量内部团队使用，尤其适合那些没有进入大型企业系统、但又需要严肃管控的项目。

## 为什么需要它

很多项目一开始不是在系统里诞生的，而是在文件里诞生的。

常见情况是：

- 项目总控表里混着预算。
- 预算表里分不清估算、确认预算、实际支出。
- 传播日历里把人名、渠道、内容写在同一个格子里。
- 负责人、部门、截止日期经常缺失。
- 项目进展只存在聊天记录或会议纪要里。
- 管理者想看全局，但又不能随便修改别人的项目。

ShadowPM 的目标不是让用户重新学习一套复杂系统，而是让 AI 先完成结构化工作，人再在表格里快速确认和补齐。

## 核心工作流

ShadowPM 当前的核心工作流是：

1. 上传 Excel、PDF、Word 或粘贴项目文本。
2. AI 识别项目信息。
3. AI 生成写入计划，说明哪些会创建、哪些需要人工确认。
4. 用户确认后创建项目。
5. 系统生成三大件：
   - 项目管控总表
   - 资金账本
   - 执行日历
6. 用户在正式表格中继续修改、补齐和管理。
7. 所有关键变更进入项目活动记录。
8. 按需导出标准工作簿、生成基于正式数据的周报/月报，或创建可撤销的只读分享链接。

## 三大核心模块

### 项目管控总表

项目管控总表是项目的主工作台。

它记录：

- 管控事项
- 负责人
- 负责部门
- 截止日期
- 状态
- 详细描述
- 进度或结论
- AI 识别置信度、来源、缺失字段和冲突信息

所有缺失字段都应该能直接在表里补齐。

### 资金账本

资金账本记录预算变化，而不是只显示一个预算数字。

它记录：

- 预算确定
- 预算增补
- 预算分配
- 实际支出
- 支出退款
- 冲正
- 流转记录

ShadowPM 的原则是：计划预算只是元信息，真正的资金状态来自预算流水。

### 执行日历

执行日历不是装饰性的月历，也不是管控事项的自动复制。

它只记录正式执行节点，例如：

- 哪一天执行
- 哪个渠道
- 什么内容
- 谁负责
- 当前状态

如果一个事项只是“进行中”，但没有明确执行日期，它不应该自动进入日历。

## 权限原则

当前 Alpha 阶段的权限规则是：

- 项目负责人默认可以编辑自己的项目。
- 管理者可以查看全局项目，但不会自动获得编辑权。
- 可编辑协作者可以修改项目。
- 只读协作者可以查看项目。
- 管控事项里的负责人字段只是业务归属，不等于系统编辑权限。

这个设计是为了避免“管理者能看所有项目”被误解成“管理者可以随便改所有项目”。

## 当前 Alpha 能做什么

当前版本已经可以：

- AI 上传生成项目
- 手动创建项目
- 编辑项目管控总表
- 管理预算流水
- 管理执行日历
- 查看项目活动
- 收集团队使用反馈
- 使用 Command Center 查询项目、预算、日历和关注项
- 查看管理者大盘
- 管理项目协作者
- 导出项目管控工作簿
- 生成有来源依据的周报和月报
- 创建可撤销、30 天失效的外部只读链接和 ICS 日历订阅
- 使用 `/api/health` 检查服务与数据库连接

## 当前 Alpha 还不是什么

它还不是：

- 完整企业级权限系统
- 完整财务系统
- 完整 Jira / ClickUp 替代品
- 完整自动报告系统
- 可公开大规模使用的生产 SaaS

当前阶段最重要的是验证核心闭环：

> 上传或创建项目 → 生成管控表/预算/日历 → 人工快速修正 → 持续管理 → 留下可追溯记录。

## 试运营想验证什么

请团队成员重点反馈：

- AI 有没有把表格结构拆对。
- 预算有没有被误判为支出。
- 日历有没有把负责人、渠道、内容拆清楚。
- 缺失字段是否容易补。
- 手动创建和修改是否顺手。
- 项目活动记录是否可信。
- 权限提示是否清楚。
- 页面是否太复杂。
- 是否愿意用它替代手工整理 Excel。

测试不是为了证明 ShadowPM 已经完美，而是为了找到它最该修的地方。
